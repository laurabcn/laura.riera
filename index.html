ADR-01 · THN (The Hotels Network) En producció
Event Sourcing amb snapshots per garantir traçabilitat per bounded context
Context

Múltiples bounded contexts necessitaven un historial fiable de canvis de negoci i la capacitat de reconstruir l'estat en qualsevol moment, sense penalitzar el rendiment de lectura a mesura que creixia el volum d'events.

Decisió

Event store amb snapshots periòdics + CQRS, separant els buses de comandos, queries i esdeveniments amb Symfony Messenger. Streaming entre contexts via Kafka; AWS SQS amb dead-letter queues per a la resiliència en consums asíncrons.

Resultat

Reconstrucció d'estat auditable per a qualsevol entitat, desacoblament real entre contexts i menys incidents de "per què les dades no coincideixen" — el sistema explica per si mateix com s'ha arribat a un estat.

ADR-02 · THN (The Hotels Network) En producció
Agents Claude Code per a auditoria de convencions i revisió de codi
Context

Mantenir consistència de convencions d'equip (PHPStan nivell 9, PSR-12, patrons DDD/hexagonal) a mesura que el codebase i l'equip creixien era cada cop més costós de revisar manualment.

Decisió

Disseny de skills i agents especialitzats amb Claude Code, amb integracions MCP, per automatitzar revisió de codi, auditoria de convencions i diagnòstic de dades abans que arribin a revisió humana.

Resultat

Menys temps de l'equip en revisions repetitives de forma/convenció, i revisions humanes centrades en disseny i lògica de negoci en lloc d'estil.

ADR-03 · Promofarma Completat
CQRS + Event Sourcing en un sistema d'alt volum
Context

Dominis de negoci complexos amb alt volum de transaccions necessitaven separar clarament lectura i escriptura per mantenir consistència sense coll d'ampolla en les consultes.

Decisió

Modelatge amb Domain-Driven Design i implementació de CQRS, amb comunicació asíncrona entre serveis via Kafka dins d'una arquitectura orientada a microserveis.

Resultat

Traçabilitat de dades garantida i lectura/escriptura escalades de forma independent.

ADR-04 · Atrápalo / Drinks and Co Completat
Refactor progressiu d'un monòlit cap a Domain-Driven Design
Context

Codebases en PHP 5, amb lògica de negoci fortament acoblada a la infraestructura i al framework, sense límits clars entre dominis, dificultaven l'evolució i el manteniment a mesura que el producte creixia.

Decisió

Procés progressiu en paral·lel a la migració de versió de PHP (5 → 7 → 8): extracció de la lògica de negoci de la infraestructura, identificació dels bounded contexts naturals dins del monòlit, i aplicació de bones pràctiques i clean code (tipat estricte, eliminació de codi mort, reducció d'acoblament) en cada mòdul migrat.

Resultat

Monòlit transformat de forma incremental cap a una arquitectura modular orientada a dominis, sobre una base de codi moderna i mantenible, sense talls de servei ni necessitat d'una reescriptura completa.

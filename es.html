ADR-01 · THN (The Hotels Network) En producción
Event Sourcing con snapshots para garantizar trazabilidad por bounded context
Contexto

Varios bounded contexts necesitaban un historial fiable de cambios de negocio y la capacidad de reconstruir el estado en cualquier momento, sin penalizar el rendimiento de lectura a medida que crecía el volumen de eventos.

Decisión

Event store con snapshots periódicos + CQRS, separando los buses de comandos, queries y eventos con Symfony Messenger. Streaming entre contexts vía Kafka; AWS SQS con dead-letter queues para la resiliencia en consumos asíncronos.

Resultado

Reconstrucción de estado auditable para cualquier entidad, desacoplamiento real entre contexts y menos incidentes de "por qué los datos no coinciden" — el sistema explica por sí mismo cómo se ha llegado a un estado.

ADR-02 · THN (The Hotels Network) En producción
Agentes Claude Code para auditoría de convenciones y revisión de código
Contexto

Mantener la consistencia de las convenciones de equipo (PHPStan nivel 9, PSR-12, patrones DDD/hexagonal) era cada vez más costoso de revisar manualmente a medida que crecían el codebase y el equipo.

Decisión

Diseño de skills y agentes especializados con Claude Code, con integraciones MCP, para automatizar la revisión de código, la auditoría de convenciones y el diagnóstico de datos antes de llegar a revisión humana.

Resultado

Menos tiempo del equipo en revisiones repetitivas de forma/convención, y revisiones humanas centradas en diseño y lógica de negocio en lugar de estilo.

ADR-03 · Promofarma Completado
CQRS + Event Sourcing en un sistema de alto volumen
Contexto

Dominios de negocio complejos con alto volumen de transacciones necesitaban separar claramente lectura y escritura para mantener consistencia sin cuello de botella en las consultas.

Decisión

Modelado con Domain-Driven Design e implementación de CQRS, con comunicación asíncrona entre servicios vía Kafka dentro de una arquitectura orientada a microservicios.

Resultado

Trazabilidad de datos garantizada y lectura/escritura escaladas de forma independiente.

ADR-04 · Atrápalo / Drinks and Co Completado
Refactor progresivo de un monolito hacia Domain-Driven Design
Contexto

Codebases en PHP 5, con lógica de negocio fuertemente acoplada a la infraestructura y al framework, sin límites claros entre dominios, dificultaban la evolución y el mantenimiento a medida que crecía el producto.

Decisión

Proceso progresivo en paralelo a la migración de versión de PHP (5 → 7 → 8): extracción de la lógica de negocio de la infraestructura, identificación de los bounded contexts naturales dentro del monolito, y aplicación de buenas prácticas y clean code (tipado estricto, eliminación de código muerto, reducción de acoplamiento) en cada módulo migrado.

Resultado

Monolito transformado de forma incremental hacia una arquitectura modular orientada a dominios, sobre una base de código moderna y mantenible, sin cortes de servicio ni necesidad de una reescritura completa.
